# Database Schema

## Enums

```prisma
enum Role {        // Message role
  USER
  ASSISTANT
  ERROR
}

enum Mode {        // Session mode
  BUILD
  PLAN
}

enum MessageStatus {
  COMPLETE         // Finished successfully
  INTERRUPTED      // Stopped mid-stream
}
```

## Session

```prisma
model Session {
  id        String    @id @default(cuid())
  UserId    String                         // Currently hardcoded as "mock-user"
  title     String                         // Auto-generated from first message
  cwd       String?                        // Working directory context
  createdAt DateTime  @default(now())
  UpdatedAt DateTime  @updatedAt
  messages  Message[]
  @@index([UserId])
}
```

## Message

```prisma
model Message {
  id        String        @id @default(cuid())
  SessionId String                         // Foreign key to Session
  role      Role                           // USER | ASSISTANT | ERROR
  status    MessageStatus                  // COMPLETE | INTERRUPTED
  model     String                         // Model ID (e.g., "claude-opus-4-6")
  content   String                         // Full text content
  parts     Json?                          // Structured parts (reasoning, tool-calls, text)
  mode      Mode                           // BUILD | PLAN
  duration  Int?                           // Response time in seconds
  createdAt DateTime      @default(now())
  session   Session       @relation(fields: [SessionId], references: [id], onDelete: Cascade)
  @@index([SessionId])
}
```

## Key Design Notes

- `parts` (JSON field) is for structured message content — reasoning steps, tool calls, and text segments. Currently not populated; content is stored in the flat `content` string.
- `duration` is in **seconds** (rounded from milliseconds in the server).
- `INTERRUPTED` status is used when a stream is aborted before completion (implementation pending — currently the server returns early on abort without saving).
- Cascade delete: deleting a Session removes all its Messages.
- The schema is generated by Prisma 7.8.0 with the `prisma-client` generator and `@prisma/adapter-pg` for PostgreSQL.

## Migration Workflow

```bash
cd packages/database
bunx prisma generate   # Regenerate client after schema changes
```

The generated client lives in `packages/database/generated/prisma/`.
